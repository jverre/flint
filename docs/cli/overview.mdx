The `flint` CLI groups commands under daemon control, backend selection, and agent deployment. Most of the time you'll just need `flint start` to get the daemon running and `flint app` to open the TUI.

## Core commands

| Command | Description |
|---------|-------------|
| [`flint start`](/cli/start) | Start the daemon (`--backend`, `--port`, `--data-dir`, `--state-dir`) |
| [`flint app`](/cli/app) | Open the interactive TUI |
| [`flint list`](/cli/list) | List running sandboxes |
| [`flint stop <vm_id>`](/cli/stop) | Kill a sandbox |

## Backends

| Command | Description |
|---------|-------------|
| [`flint backends list`](/cli/backends) | List installed backends and their preflight status |

## Agents

| Command | Description |
|---------|-------------|
| [`flint agents list`](/cli/agents#list) | List available pre-packaged agents |
| [`flint agents info <name>`](/cli/agents#info) | Show agent details |
| [`flint agents build <name>`](/cli/agents#build) | Pre-build an agent template for instant deploys |
| [`flint agents deploy <name>`](/cli/agents#deploy) | Deploy an agent in a microVM |

## Host setup

| Command | Description |
|---------|-------------|
| `flint install-deps` | Install Firecracker, jailer, and vmlinux kernel (Linux) |
| `flint install-deps --check` | Verify installed dependencies |
| `flint setup-macos` | Prepare macOS Virtualization.framework guest assets |
| `flint setup-macos --check` | Verify macOS guest assets |

## Configuration

The daemon reads configuration from environment variables:

| Variable | Default | Description |
|----------|---------|-------------|
| `FLINT_PORT` | `9100` | Port the daemon listens on |
| `FLINT_DATA_DIR` | `/microvms` (Linux), `~/Library/Application Support/flint/data` (macOS) | Where VM rootfs images and snapshots are stored |
| `FLINT_STATE_DIR` | `/tmp/flint` (Linux), `~/Library/Application Support/flint/state` (macOS) | Where the daemon keeps its state (SQLite db, pid file, logs) |
| `FLINT_BACKEND` | host default | Backend plugin to use (`firecracker`, `cloud-hypervisor`, `macos-vz`) |

```bash
FLINT_PORT=9200 FLINT_BACKEND=cloud-hypervisor flint start
```
