---
title: "Templates"
description: "Build custom sandbox environments from OCI images."
---

By default, sandboxes use a minimal Alpine Linux rootfs pulled from
`ghcr.io/jverre/flint/base:latest`. If you need Python packages, system
libraries, or a specific runtime pre-installed, you can build a custom
template from any OCI image.

Templates work by pulling an OCI image directly from a registry (using
[crane](https://github.com/google/go-containerregistry) — no Docker daemon),
extracting the flattened filesystem into an ext4 image, injecting the Flint
guest agent, and creating a golden snapshot from it. Once built, you can
create sandboxes from your template just like the default one — with the
same fast startup time.

## Building a template

<Tabs>
  <Tab title="Python SDK">
    ```python
    from flint import Template, Sandbox

    template = (Template("my-python-env")
        .from_oci_image("python:3.11-slim")
        .build())

    sandbox = Sandbox(template_id=template.template_id)
    result = sandbox.run_code("import numpy; print(numpy.__version__)")
    ```

    The `Template` class uses a fluent builder - each method returns `self` so you can chain calls.

    **Constructor**

    <ParamField path="name" type="str" required>
      Human-readable name for this template.
    </ParamField>

    <ParamField path="rootfs_size_mb" type="int" default="500">
      Size of the rootfs image in MB. Increase this if your base image is large.
    </ParamField>
  </Tab>
  <Tab title="REST API">
    `POST /templates/build`

    ```bash
    curl -X POST http://localhost:9100/templates/build \
      -H "Content-Type: application/json" \
      -d '{
        "name": "my-python-env",
        "image_ref": "python:3.11-slim",
        "rootfs_size_mb": 500
      }'
    ```

    <ParamField body="name" type="string" required>
      Human-readable name.
    </ParamField>

    <ParamField body="image_ref" type="string" required>
      OCI image reference (e.g. `python:3.11-slim`, `ghcr.io/org/img:tag`).
    </ParamField>

    <ParamField body="rootfs_size_mb" type="int" default="500">
      Rootfs image size in MB.
    </ParamField>

    <ParamField body="inject_flint" type="bool" default="true">
      When `false`, the image is assumed to already contain `flintd` and
      `/etc/init-net.sh` (e.g. pre-built Flint base images).
    </ParamField>

    **Response** `200` - returns immediately, build happens in background

    ```json
    {"template_id": "tpl_abc123", "status": "building"}
    ```

    Poll `GET /templates/{template_id}` until `status` is `"ready"`.
  </Tab>
</Tabs>

## Base image methods

Pick one:

```python
.from_oci_image("python:3.11-slim")                 # any OCI reference
.from_oci_image("ghcr.io/org/custom:latest",
                inject_flint=False)                 # pre-built Flint image
.from_ubuntu_image("22.04")                         # shorthand for ubuntu:22.04
.from_python_image("3.11-slim")                     # shorthand for python:3.11-slim
.from_node_image("20-slim")                         # shorthand for node:20-slim
.from_alpine_image("3.19")                          # shorthand for alpine:3.19
```

To customise a base image (install packages, add files, set env vars),
build your own OCI image and push it to any registry — then reference it
with `.from_oci_image(...)`. You can use GitHub Actions, Buildkit, Kaniko,
or any OCI image builder; Flint only needs the final image to be pullable.

## TemplateInfo

`build()` blocks until the template is ready and returns a `TemplateInfo`:

```python
info.template_id  # "tpl_abc123" - use with Sandbox(template_id=...)
info.name         # "my-python-env"
info.status       # "ready"
```

## Caching

Pulls are cached locally by image digest under `{FLINT_DATA_DIR}/.oci-cache`.
Building the same `image_ref` twice hits the cache on the second call and
skips the network fetch. The cache is LRU-evicted once it exceeds 20 entries.

## A note on build time

First build of a template pulls the image and creates a golden snapshot
— usually 10-30 seconds depending on image size. Cached rebuilds are near
instant. Once built, `Sandbox(template_id=...)` is just as fast as the
default template.
