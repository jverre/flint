## Prerequisites

- **Linux** with KVM enabled (`/dev/kvm` must exist) — or **macOS** on Apple Silicon with the Virtualization entitlement
- Python 3.12+
- Root access (for installing binaries and managing VMs on Linux)

## Installation

<Steps>
  <Step title="Install Flint">
    ```bash
    pip install flint
    ```

    Or install from source:

    ```bash
    git clone https://github.com/jverre/flint
    cd flint
    uv sync
    ```
  </Step>

  <Step title="Install a hypervisor (Linux only)">
    Flint ships with two Linux backend plugins: **Firecracker** and **Cloud-Hypervisor**. Install either (or both) — you pick which one runs at daemon startup.

    ```bash
    # Firecracker — ships with the install-deps helper (also installs the vmlinux kernel)
    sudo flint install-deps

    # Cloud-Hypervisor — download the release binary
    curl -fsSL -o /tmp/ch https://github.com/cloud-hypervisor/cloud-hypervisor/releases/download/v46.0/cloud-hypervisor-static
    sudo install -m 755 /tmp/ch /usr/local/bin/cloud-hypervisor
    ```

    On macOS, skip this step — `flint setup-macos` below handles it.
  </Step>

  <Step title="Build the base rootfs">
    On Linux:

    ```bash
    sudo bash setup-rootfs.sh
    ```

    On macOS:

    ```bash
    flint setup-macos
    ```

    You only need to do this once.
  </Step>

  <Step title="Start the daemon">
    The daemon manages all VM lifecycle in the background. It listens on `localhost:9100` by default and auto-selects a backend based on your host:

    ```bash
    sudo flint start
    ```

    To pick a specific backend:

    ```bash
    sudo flint start --backend cloud-hypervisor
    # or: FLINT_BACKEND=cloud-hypervisor sudo flint start
    ```

    You should see it print `Golden snapshot ready` (Firecracker) or the equivalent ready marker for your backend.
  </Step>

  <Step title="Run your first sandbox">
    In a new terminal:

    ```python
    from flint import Sandbox

    sandbox = Sandbox()
    result = sandbox.commands.run("echo hello from a microVM")
    print(result.stdout)  # hello from a microVM
    sandbox.kill()
    ```
  </Step>
</Steps>

## What just happened

When you called `Sandbox()`, the daemon:

1. Picked the configured backend (Firecracker, Cloud-Hypervisor, or macOS-VZ)
2. Grabbed a pre-copied rootfs from the pool
3. Set up a network namespace with a TAP device (Linux) or internal network (macOS)
4. Booted the VM — either by loading the golden snapshot (Firecracker) or doing a fast fresh boot (Cloud-Hypervisor, macOS-VZ)
5. Waited for the guest agent (`flintd`) to be healthy

The whole thing took 50–150ms on Linux and a few hundred ms on macOS. The `commands.run()` call then proxied your shell command through `flintd` inside the VM.
